# Day 11 – Blackjack Game

## Overview

This project is a text-based Blackjack game built with Python. The player competes against the computer by drawing cards and trying to get as close as possible to 21 without going over.

The project focuses on applying Python fundamentals learned in previous days, especially functions, loops, conditionals, lists, and program flow.

## Topics Covered

* Functions and function calls
* Parameters and return values
* Lists and list manipulation
* Random module
* While loops
* For loops
* Conditional statements (`if`, `elif`, `else`)
* Boolean variables
* Score calculation
* Game logic and flow control

## What I Learned

* How to create reusable functions to avoid repeating code.
* How to use the `random` module to generate random cards.
* How to store multiple values using lists.
* How to calculate scores using functions and return the result.
* How to handle special cases like an Ace (`11`) becoming a `1` when the score goes over 21.
* How to control a game's flow using loops.
* How to separate different parts of a program into smaller functions.
* How to compare values and determine the winner based on different conditions.

## Project Features

* Random card dealing system.
* Player can choose to draw another card or pass.
* Computer automatically draws cards until reaching a score of at least 17.
* Blackjack detection.
* Winner comparison between player and computer.
* Option to restart the game.

## Challenges

The biggest challenge was understanding how to organize the game logic.

At first, it was difficult to understand where loops, conditions, and functions should be placed. Breaking the project into smaller parts helped me understand the flow:

1. Deal cards.
2. Calculate scores.
3. Handle the player's turn.
4. Handle the computer's turn.
5. Compare final scores.
6. Restart the game if the player wants.

## Reflection

This project helped me understand how multiple Python concepts work together in a real application.

I learned that programming is not only about writing code but also about planning the structure of the program before building it.

The Blackjack project improved my confidence in using functions, loops, and conditional logic to create interactive programs.
