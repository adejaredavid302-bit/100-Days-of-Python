# Day 9 – Secret Auction

## Overview

This project is my solution to the Secret Auction challenge from Day 9 of Angela Yu's **100 Days of Code: Python Bootcamp**.

The program collects bids from multiple users, stores them in a dictionary, and determines the highest bidder using a custom algorithm.

- Key-Value Pairs
- Functions
- Function Parameters
- While Loops
- For Loops
- Conditional Statements
- User Input
- Variables
- Algorithms

## What I Learned

- How to create and use dictionaries.
- How to add key-value pairs dynamically.

- How to write reusable functions with parameters.
- How function parameters make code more flexible and reusable.
- How to find the highest value in a dictionary without using Python's built-in `max()` function.
- How to separate data collection from data processing by using functions.

## Challenges

The most challenging part of this project was figuring out how to find the highest bidder. My first approach stored the data incorrectly, so I redesigned my dictionary to use the bidder's name as the key and the bid as the value.

Instead of relying on Python's built-in `max()` function, I implemented my own algorithm to compare bids one by one. This helped me understand the logic behind finding the largest value in a collection.

I also explored different ways of iterating through dictionaries before deciding on the approach I preferred.

## Reflection

Day 9 completely changed how I think about dictionaries. At first, they were confusing because they use keys instead of indexes like lists. By completing this project, I became much more comfortable storing, accessing, and processing data with dictionaries.

This project also helped me understand how breaking a program into smaller functions makes the code cleaner, easier to read, and easier to maintain.
