# Day 13 – Debugging

## Overview

Day 13 focused on one of the most important programming skills: **debugging**. Instead of building a new project, this lesson emphasized identifying, understanding, and fixing errors in existing code.

The goal was to develop a systematic approach to finding bugs rather than guessing or randomly changing code until it worked.

## Topics Covered

* Debugging techniques
* Reading Python error messages
* Syntax errors
* Runtime errors
* Logical errors
* Using `print()` statements for debugging
* Tracing program execution
* Problem-solving strategies

## What I Learned

* How to read and understand Python error messages.
* The difference between syntax, runtime, and logical errors.
* How to trace the flow of a program to locate bugs.
* How to inspect variables using `print()` statements.
* Why changing one thing at a time makes debugging easier.
* The importance of understanding the cause of an error before attempting to fix it.

## Key Debugging Principles

* Describe the problem before trying to solve it.
* Reproduce the bug consistently.
* Read error messages carefully.
* Use `print()` statements to inspect variables and program flow.
* Test small sections of code instead of the entire program.
* Fix one issue at a time.
* Test the program again after every fix.

## Challenges

The biggest challenge was resisting the urge to make random changes when something didn't work. Instead, I learned to slow down, analyze the problem, and debug the code step by step.

## Reflection

Day 13 taught me that debugging is a fundamental programming skill. Writing code is only part of software development—the ability to identify and fix problems is equally important.

Learning a structured debugging process has made me more confident when working on Python projects, and I now understand that encountering bugs is a normal part of programming.
